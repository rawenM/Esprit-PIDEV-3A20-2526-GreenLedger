CAUSE :
Google reCAPTCHA v2 ne fonctionne pas bien dans JavaFX WebView.
C'est une limitation technique connue de JavaFX.

Le WebView de JavaFX :
- Ne supporte pas tous les JavaScript modernes
- A des problèmes avec les iframes de Google
- Rend reCAPTCHA très lent ou non fonctionnel

═══════════════════════════════════════════════════════════════
   🔧 POURQUOI reCAPTCHA EST LENT ?
═══════════════════════════════════════════════════════════════

Problèmes techniques de JavaFX WebView :

1. JavaScript limité
   - WebView utilise WebKit ancien
   - Ne supporte pas tous les JS modernes
   - reCAPTCHA utilise du JS avancé

2. Iframes problématiques
   - reCAPTCHA charge dans un iframe
   - JavaFX a des bugs avec les iframes
   - Communication iframe lente

3. Réseau
   - Chaque vérification appelle l'API Google
   - Latence réseau
   - Timeout fréquents

4. Rendu
   - WebView rend mal certains éléments
   - Images reCAPTCHA lentes à charger
   - Interactions souris problématiques
