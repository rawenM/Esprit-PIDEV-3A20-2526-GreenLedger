                          KEY SYSTEM FEATURES
═══════════════════════════════════════════════════════════════════════════════

🔒 ATOMIC TRANSFERS
   What: All-or-nothing credit movement
   How: Database row locks + transaction rollback
   Why: Prevents partial transfers, ensures data consistency
   Guarantee: Debit source OR add destination (never partial)

📊 EVENT SOURCING  
   What: Immutable event log with blockchain-ready hashing
   How: SHA-256 chain with previous_hash links
   Why: Full audit trail for compliance & provenance
   Guarantee: Tampering breaks the chain

💰 CREDIT CONSERVATION
   What: Total credits never change
   Equation: Available + Retired = Constant
   Why: Mathematical proof of no loss
   Verification: SELECT SUM(available_credits + retired_credits)

🔐 ISOLATION
   What: Test system independent of projet code
   Why: No breaking changes to existing functionality
   How: Zero imports from Controllers.* or projet classes
   Benefit: Can test independently
